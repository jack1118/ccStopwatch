## 1. 目的與背景

田徑場跑班教練在**同一堂課同時帶多組**不同配速的學員。教練需要一支能在 iPhone 上、**同畫面同時計時多組**、依 Nike NRC 配速顏色區分、並能在課後檢視各組每趟分段成績的碼表。

市場調研結論：App Store 上**沒有**任何現存維護中的 app 同時做到「NRC 顏色分組同畫面計時 ＋ 各組不同趟數的課表 ＋ 課後分段折線圖」。這是真實空缺，需自製。

### 平台決定

採 **PWA（漸進式網頁 App）**：HTML/JS 製作，iPhone Safari 開啟後「加入主畫面」即如原生 app，可離線、可存檔。原因：開發機為 Windows，無法編譯原生 SwiftUI；PWA 可在 Windows 完整開發完成並裝到 iPhone，是唯一務實路徑。

## 2. 核心概念與名詞

| 名詞 | 定義 |
|---|---|
| **課程 / Session** | 一堂課。含一份共用課表、一組組別清單、所有計時資料。可存檔、可重開。 |
| **組別 / Group** | 一個配速組，對應一支獨立碼表。屬性：NRC 顏色、組號、（可覆寫的）趟數、可選目標配速、可選學員名單。 |
| **課表 / Plan** | 全班共用的訓練內容，由**多個段落**組成。 |
| **段落 / Segment** | 課表的一段，如「400m × 6 趟，間休 90s」。屬性：標籤/距離、趟數、每趟間休息秒數。 |
| **趟 / Rep（圈）** | 一個段落內跑一次。每趟記錄一個**跑步分段時間**與一個**實際休息秒數**。 |

### NRC 顏色（由快到慢，固定順序）

黃 `#E8B800` ＞ 黑 `#1C1C1E` ＞ 紫 `#7B3FE4` ＞ 藍 `#0A6CFF` ＞ 綠 `#1FA84A` ＞ 紅 `#E0392B`

組別標示為「顏色 · 第 N 組」，例：黃·1組、黑·2組。同色可多組（黃·1組、黃·3組）。

## 3. 主要畫面與流程

四個主畫面：**課程清單 → 課程設定 → 計時主畫面 → 結果圖表**。

```
課程清單 ──新增──▶ 課程設定 ──開始上課──▶ 計時主畫面 ──結束──▶ 結果圖表
   ▲  └──點既有課程──────────────────────────┘（可重開任何狀態）        │
   └────────────────────────────────────────────────────────────────┘
```

### 3.1 課程清單（首頁）

- 列出所有已存課程卡片：名稱、日期、組數、狀態（進行中/已完成）。
- 點卡片進入該課程（計時中 → 計時主畫面；已完成 → 結果圖表）。
- 「＋ 新課程」進入課程設定。
- 每張卡片可重新命名、刪除、匯出 CSV。

### 3.2 課程設定

- **課程名稱**（預設今日日期）。
- **共用課表編輯**：可新增多個段落，每段填「標籤/距離、趟數、間休秒數」。例：`400m ×6 休90s` → `200m ×4 休60s`。課表**可留空**（純碼表）。
- **新增組別**：點 NRC 顏色色塊即新增一組，自動給「該色下一個組號」。每組可：
  - 覆寫趟數（如綠/紅只跑 8 趟）。
  - 可選設定目標配速（秒/趟）。
  - 學員名單**可現在或課後再填**。
- 「開始上課」進入計時主畫面。設定隨時可回來改。

### 3.3 計時主畫面（核心）

**版面：自動切換**
- 2–4 組 → 直向堆疊大卡（計時字最大、最好按）。
- 5 組以上 → 雙欄網格（六組一畫面、不需捲動）。

**每組為一張卡片，獨立狀態機**（無「全部開始」，各組分開起跑）：

```
未開始 ──點開始──▶ 跑步中 ──點卡片(記圈)──▶ 休息中 ──點出發──▶ 跑步中(下一趟)
                                                  │                    │
                                              (趟數跑完 / 點⏹)─────────▶ 完成
```

| 狀態 | 卡片顯示 | 點卡片主體的動作 |
|---|---|---|
| **未開始** | 大「▶ 開始」按鈕 + 課表摘要 | 開始第 1 趟（起跑、跑步計時歸零開跑） |
| **跑步中** | 當趟計時（最大）、第 N 趟、上一趟時間、下一步（休息秒數/距離） | **記這趟成績**，進入休息中 |
| **休息中** | 休息正數計時、進度條對到目標休息、上一趟成績 | **出發**下一趟（記錄實際休息秒數，跑步計時歸零開跑） |
| **完成** | 變淡、顯示總時間 | 點進該組結果圖表 |

**卡片資訊大小優先序**（依需求）：當趟計時 ＞ 第幾趟 ＞ 上一趟時間 ＞（有課表才顯示）下一步是什麼/休息。

**計時顯示細節**：
- **精準到整秒，不顯示小數**，格式 `m:ss`（如 `1:12`）。
- **秒為視覺焦點**：「秒」的字級明顯大於「分」（分較小、置於前），跑步中的當趟計時放到最大、撐滿卡片空白區（網格約 50px、2–4 組大卡約 100px，依空間自動撐大）。
- **讀秒跳動**：每跳一秒，秒數做一次輕微放大（約 +16%）再回正，營造讀秒節奏感。各組起跑時間不同，故跳動不同步。
- **休息超時不變色**：卡片永遠維持該組 NRC 顏色（綠仍是綠），超時僅以紅色 `+N s` 標記與紅色進度條提示，不重塗整張卡片，以免破壞組別顏色身分。

**彈性休息**（關鍵）：休息為**正數計時**，原定休息秒數只是參考。到點時嗶一聲＋震動提醒一次，之後**繼續累加**並轉紅、顯示超出秒數（如 `+0:14`）。下一趟一律由教練點「出發」才開始 —— 計時器不自動跳趟。

**無課表（純碼表）情形**：若課表留空或該段未設休息秒數，則沒有休息階段 —— 跑步中點卡片直接記這趟並立即開始下一趟（連續按圈模式），不顯示休息倒數。此情形無「目標趟數」，不會自動完成，僅能以 `⏹ 結束` 收尾。

**每張卡片角落控制**：
- `↶ 撤銷`：撤銷該組剛剛的誤觸（誤記的圈 / 誤點的出發）。
- `⏹ 結束`：手動結束該組。

**全域行為**：
- 進入計時主畫面即請求 **Screen Wake Lock**，避免螢幕休眠（田徑 app 最大痛點）。
- 計時以**時間戳記差值**計算（非每秒累加），背景/重新整理後回前景仍正確。
- 所有狀態變更即時寫入儲存，意外關閉可復原。

### 3.4 結果圖表

- **疊圖比較**：各組每趟秒數折線圖，各自 NRC 顏色，x=趟、y=秒。圖例可點選只看某幾組。
- **目標配速參考線**：有設目標的組，圖上畫該色虛線。
- **單組詳細檢視**：點一組進放大版 —— 每趟長條（跑步分段）＋該趟實際休息秒數，標出最佳/最慢趟。
- **明細表**：各組各趟秒數＋平均＋實際休息；趟數多可左右捲、組數多可上下捲。
- **匯出**：CSV（給 Excel）、整頁截圖分享（PNG）。

### 3.5 學員名單（課後可補）

- 任一組可在課中或課後輸入學員名稱（純文字清單，附在該組）。
- 顯示於組別與結果，不影響計時（一組一碼表）。

## 4. 資料模型（持久化）

以 JSON 存於瀏覽器（`localStorage`，鍵為課程 id；課程清單存一份索引）。資料量小（數堂課、每組數十趟）。

```
Session {
  id, name, createdAt, status: "active" | "done",
  plan: { segments: [ { id, label, distance?, reps, restSec } ] },
  groups: [ Group ],
}
Group {
  id, color: "yellow"|"black"|"purple"|"blue"|"green"|"red", number,
  repsOverride?: number,        // 不填則用課表趟數加總
  targetPaceSec?: number,
  athletes: [ string ],
  state: "idle"|"running"|"resting"|"done",
  runStartTs?, restStartTs?,    // 時間戳，供精準與復原
  reps: [ { index, runSec, restSec } ]   // 已完成的每趟，秒數為整數秒
}
```

> 設計重點：計時資料以**組別槽位**記錄，與學員身分**解耦** —— 名單可事後補，不影響已記錄的趟數資料。

## 5. 技術架構

- **框架**：React + TypeScript + Vite。`vite-plugin-pwa` 產生 manifest + service worker（離線、可安裝）。
- **圖表**：手刻 SVG 折線/長條（無重依賴），維持 bundle 輕量、樣式可控。
- **計時核心**：單一 `requestAnimationFrame`/`setInterval` tick，每組顯示值由 `now - startTs` 即時計算。各組狀態與 tick 邏輯獨立、可單獨測試。
- **儲存層**：薄封裝 `localStorage`（`load/save/list/remove`），與 UI 解耦。
- **螢幕喚醒**：Screen Wake Lock API（iOS 16.4+ Safari 支援），失敗時降級不報錯。
- **提示**：休息到點用 Web Audio 嗶聲 + `navigator.vibrate`（iOS 對 vibrate 支援有限，故聲音為主、震動為輔）。

### 模組邊界

| 模組 | 職責 | 依賴 |
|---|---|---|
| `timer/` | 純函式：由時間戳算 elapsed、狀態轉換 | 無（純邏輯，可單測） |
| `storage/` | 課程讀寫 localStorage | 無 |
| `screens/SessionList` | 課程清單 | storage |
| `screens/SessionSetup` | 課表＋組別設定 | storage |
| `screens/Timer` | 計時主畫面、卡片、版面自動切換 | timer, storage |
| `screens/Results` | 圖表、明細、匯出 | storage, chart |
| `chart/` | SVG 折線/長條 | 無 |
| `export/` | CSV、PNG 截圖 | 無 |

## 6. 交付與安裝到 iPhone

1. `/goal` 在 Windows 上完成開發，`npm run build` 產生靜態檔。
2. 部署到 HTTPS 靜態主機（GitHub Pages / Netlify / Vercel 擇一；PWA 安裝需 HTTPS）。
3. iPhone Safari 開該網址 → 分享 → 「加入主畫面」→ 全螢幕離線使用。

> 開發期間可用 `npm run dev` 在電腦瀏覽器測試；行動裝置驗證在部署後進行。

## 7. 測試策略

- **單元測試**（`timer/`、`storage/`、`export/`、`chart/` 座標計算）：純函式，覆蓋狀態轉換、休息超時、趟數覆寫、CSV 內容。
- **元件測試**：卡片在各狀態的顯示與點按行為；版面 2–4 組 vs 5+ 組切換。
- **手動驗證清單**（部署後 iPhone）：多組同時計時、休息超時提醒、撤銷、螢幕不休眠、離線開啟、重整復原、匯出。

## 8. 範圍界線（YAGNI）

**納入**：上述 1–7 全部。

**明確不做**（本版）：
- 組內個別學員計時（一組一碼表）。
- 雲端同步 / 多裝置（純本機 localStorage）。
- Apple Watch companion。
- 音量鍵按圈（瀏覽器無法攔截，改用大面積點按）。
- 帳號/登入、社群分享、線上排行。
