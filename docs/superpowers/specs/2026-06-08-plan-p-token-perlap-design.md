# 課表 `p` token 改為「每圈」語意 + 配速雙義

日期：2026-06-08
狀態：已核可，待實作

## 背景與問題

跑班慣例：說「p104」一定指**每圈** 104 秒。但目前程式碼把 `p` 當成「跑完該距離的總秒數」：

- `planText.ts:66-68`：`p<數字>` 直接存成 `targetSec`（item 整段距離的目標秒）。
- 因此 `1200m p104` 被讀成「1200m 跑 104 秒」（不合理），而非「每圈 104 秒」。
- 註解本身矛盾：`:17` 寫「p＝完成該距離目標秒」，`:84` 寫「p＝每圈配速秒」。實作與後者（也是使用者直覺）不一致 —— 是既有 bug。
- UI（`SessionSetup.tsx:100-101`）目標切換鈕「以距離／以每圈」預設為 `'dist'`，與「p＝每圈」的慣例衝突。

同時要讓 `p` 也能表達**每公里配速**（如 `p4'50`、`p500`），用 300 當分界。

## 目標

1. 課表文字中 `p` 永遠＝每圈秒數（不可違背）。
2. `p` 也能表配速（每公里），與每圈秒數用規則區分。
3. UI 目標預設改「以每圈」；逐組自訂欄也改「每圈」。
4. 序列化 round-trip 正確（避免 `p<總秒>` 被讀回成每圈而放大）。

## 解析規則（`p` token）

判斷順序：

1. **含 `'` 或 `:`**（`p4'50`、`p4:50`）→ **一定是配速** mm:ss/km，與數值大小無關。
2. 純數字 `p<n>`：
   - `n ≥ 300` → 配速：末兩位＝秒、其餘＝分（`450`→4:50、`500`→5:00、`615`→6:15）。秒須 `< 60`，否則該段視為無效（整串當純文字課名，回 `null`）。
   - `n < 300` → **每圈秒數**。
3. 換算成 `targetSec`（資料模型不變，仍存 item 整段目標秒）：
   - 每圈：`targetSec = round(每圈 × 距離 / lapMeters)`（例：p96 在 1200m/400m → 96×1200/400＝288）
   - 配速：`paceSecPerKm = 分×60 + 秒`、`targetSec = round(paceSecPerKm × 距離 / 1000)`，並寫入 `paceSecPerKm`。
4. `@m:ss` 與 `@p…` 維持原樣（向後相容）。`pace` 與 `p` 同時出現仍視為衝突 → 回 `null`（保留 `planText.ts:59` 現有行為）。

### 邊界與安全帶
- `p300` → 配速 3:00/km（≥300 歸配速）。
- 每圈實際範圍約 60–200 秒，恆 <300；配速 mm:ss 恆 ≥300（含 3:30＝330）。理論模糊帶 230–299（≒ 2:30–2:59/km）對跑班為菁英速度、實務空集合 → 不處理。
- 帶 `'`/`:` 一律配速，提供無歧義寫法。
- **非整圈距離的 ±1 秒重序化漂移（已接受的權衡）**：每圈↔整段兩端都用 `round`，在非 100m 對齊的距離（如 400m 場地的 300m）可能讓「每圈 90 → 重開顯示 91」這類 ≤1 秒位移。位移不累積（再次序列化即穩定）、且 timer 實際使用的 `targetSec` 不受影響。整數圈（400/800/1200…）與 100m 倍數（500/600…）皆無漂移。

## 序列化（`segLabel` / `planSummary`）

新增 `lapMeters` 參數（4 個呼叫點都拿得到：`Timer.tsx:82`、`SessionSetup.tsx:104`、`SessionSetup.tsx:251`、`ShareCard.tsx:36`）。

- 有 `paceSecPerKm` → 輸出 `@m:ss`（不變）。
- 純每圈目標 → 輸出 `p<每圈秒>`，其中 `每圈秒 = round(targetSec × lapMeters / 距離)`（取代現在的 `p<targetSec>`）。
- round-trip：`1200m`/`targetSec=288`/400m場地 → 輸出 `p96` → 讀回 `96×1200/400=288` ✓。

## UI（`SessionSetup.tsx`）

1. `modeOf` 預設由 `'dist'` 改 `'lap'`（`:101`）。
2. 逐組自訂的「距離目標」欄（`:372-375`）改為「每圈目標 · 秒/圈」：
   - 顯示 `round(targetFor(c,it) × lapMeters / 距離)`。
   - 輸入時換回絕對 `targetSec = round(每圈輸入 × 距離 / lapMeters)` 存入 `segTarget`。
   - gap 累加已含在 `targetFor`，照轉即可，不需另外處理。
